你好，欢迎来到生成式人工智能与模型基础课的第二课。今天我们讲 Embedding，也就是嵌入：模型如何把意义放进一个可以计算的空间。

上一课，我们知道语言模型接收的不是完整的词，而是 token。模型每次根据已有 token 预测下一个 token。但这里缺了一座桥：计算机最终只会处理数字，token 怎样变成模型能计算的数字？为什么模型会觉得猫和狗相近，却觉得猫和数据库连接池很远？这就是 Embedding 要解决的问题。

先从最朴素的方法开始。假设词表只有四个 token：猫、狗、汽车、香蕉。我们可以给每个 token 一个只有自己位置是 1、其他位置都是 0 的向量。猫是 1、0、0、0，狗是 0、1、0、0。这叫 one-hot 编码。

它能准确点名每个 token，但有两个问题。第一，词表如果有十几万个 token，向量就有十几万维，而且几乎全是零。第二，更重要的是，它没有表达任何关系。猫和狗的距离，跟猫和汽车的距离完全一样。编号只是在点名，不是在描述意义。

Embedding 可以先理解为一张可以学习的坐标表。表的每一行对应一个 token，每一行都是一个稠密向量。模型收到 token id 后，第一步其实只是查表，取出对应的那一行。

如果词表有十万个 token，每个向量有四千零九十六维，这张表就包含大约四点一亿个可训练参数。这些坐标不是程序员手写的。训练开始时，它们接近随机；模型看过大量文本后，为了更准确地预测下一个 token，梯度下降会不断调整坐标。在相似上下文中出现的 token，会逐渐形成相似的结构。

所以，猫和狗可能靠近；巴黎到法国的关系，与东京到日本的关系，可能呈现相似的方向。但要注意，不是某一个维度单独代表动物，另一个维度单独代表城市。意义通常分布在很多维度的组合和相对关系里。

那么，怎样判断两个向量是否相似？最常见的是余弦相似度。你可以把每个向量想成从原点射出的一支箭。余弦相似度主要看两支箭的方向是否一致，而不是长度是不是一样。接近 1，说明方向很像；接近 0，说明大致无关；接近负 1，说明方向相反。

为什么看方向？因为向量长度可能混入词频、置信度或者其他因素，而方向更适合比较结构是否相似。语义搜索就是这样工作的：先把用户问题和文档都变成向量，再找方向最接近的文档。

但距离近不等于事实正确。Embedding 学到的是训练数据里的统计关系，也会继承数据里的偏见。它还受到模型、语言、上下文和训练目标影响。不同模型产生的向量通常不能直接混在一起比较。

接下来有一个关键问题：同一个 token 是不是永远只有同一个向量？在刚进入模型、查坐标表的时候，是的。可是经过 Transformer 的多层 Attention 和前馈网络后，它会不断吸收上下文。

比如，“苹果发布了新芯片”和“我吃了一个苹果”。刚进模型时，苹果可能来自同一行坐标；经过上下文计算后，两个苹果的表示会走向不同的位置。第一个吸收了发布和芯片的信息，第二个吸收了吃的信息。

所以我们要区分初始的 token embedding，和经过 Transformer 后的上下文表示。现代检索用的 Embedding 模型，通常不是简单取输入坐标表的一行，而是对上下文化后的多个 token 表示进行汇总，得到整段文本的向量。

Embedding 和上一课的 Attention 又是什么关系？Attention 里的 Query、Key 和 Value，并不是从 token 编号直接产生的，而是从 token 的向量表示经过线性变换得到。没有向量，就不能做点积，也不能判断当前 token 应该关注上下文里的谁。

Embedding 也是 RAG，也就是检索增强生成的入口。一个典型流程分五步。第一，把文档切成片段。第二，用同一个 Embedding 模型把片段编码成向量，存进向量数据库。第三，把用户问题也编码成向量。第四，做近邻搜索，找出方向最相似的文档片段。第五，把这些片段交给大模型生成答案。

这里最常见的工程问题，往往不是向量数据库选错了。而是文档切块不合理；问题和文档用了不匹配的模型；升级了 Embedding 模型，却没有重建旧索引；或者只看相似度，没有做重排和权限过滤。

我们用一个二维例子建立直觉。假设猫的坐标是零点九、零点八；狗是零点八、零点九；汽车是负零点八、零点二。猫和狗都指向右上方，方向很接近。汽车指向左上方，和它们相差很大。

真实模型当然不止二维，可能是七百六十八、一千五百三十六或者更多维。高维空间里会出现一些反直觉现象，所以实际系统还需要专门的近似最近邻索引，在召回速度、内存和准确率之间权衡。这部分我们以后讲模型 Infra 时再展开。

最后，请记住三个判断。

第一，Embedding 不是给词分配一个神秘编号，而是学习一套可计算的相对位置。

第二，模型里的意义首先表现为关系：谁和谁靠近，哪些方向相似，以及表示如何随上下文变化。

第三，Embedding 同时连接算法和工程。向前，它连接 Attention；向外，它连接搜索、推荐、聚类和 RAG。

给你留一个开车时也能思考的问题。假设用户搜索“退款多久到账”，文档写的是“原路退回通常需要三个工作日”。关键词没有完全重合，为什么 Embedding 搜索可能找到它？反过来，为什么它也可能找到一条语义相似、但只适用于另一个国家或另一个用户等级的规则？答案是：语义相似只解决了相关性的一部分，不能替代事实条件、权限和业务规则。

下一课，我们会把视野拉高，完整拆开 Transformer：Attention 负责跨 token 获取信息，前馈网络负责逐位置加工，残差连接和归一化则帮助深层网络稳定训练。
